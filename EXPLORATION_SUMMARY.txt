CURRENT STUDENT ENROLLMENT PROCESS
================================================================================

Step 1: Student creates AdmissionApplication (form submission)
Step 2: Admin reviews application
Step 3: Admin clicks "Convert" button
Step 4: POST /api/v1/admissions/convert endpoint:
        - Creates User with role='student'
        - Auto-generates enrollmentNumber (STU[YEAR][SEQ])
        - Creates Membership record
        - Enrolls in specified batch
        - Creates Fee record with status='not_started'
        - Updates application.status to 'converted'
Step 5: Student appears in dashboard student count
Step 6: Fees tracked as payments made
Step 7: [MISSING] No way to mark completion or generate certificates

================================================================================
MULTI-INSTITUTE ARCHITECTURE
================================================================================

Key Feature: The system supports multiple institutes (schools/colleges)

Access Control:
  1. Membership Model: Links users to institutes with role per institute
  2. Middleware: getInstituteScope() - validates institute access
  3. Query Scoping: All queries filter by institute

User Roles:
  - super_admin: Access to all institutes globally
  - admin: Access to single institute only
  - instructor: Can teach in assigned batches
  - staff: Institute staff
  - student: Enrolled in batches

Multi-Institute Pattern:
  - User._id is global
  - User.institute is primary (but can have Membership to multiple)
  - Membership table tracks per-institute access
  - All queries must include institute filter

================================================================================
AUTHENTICATION FLOW
================================================================================

1. User logs in with email/password
2. NextAuth validates credentials against User.passwordHash (bcrypt)
3. JWT token issued with user._id, role, institute context
4. Subsequent requests include JWT in headers
5. Middleware extracts scope (instituteId, isSuperAdmin, userId)
6. All API endpoints validate scope before processing

  ✓ Status Requirement:

    - (Attended Classes / Total Classes) >= 75%
    - Query Attendance model

  ✓ Payment Requirement:
    - Fee.status === 'paid'
    - All installments paid

  ✓ Academic Requirement (if applicable):
    - ExamSubmission.score >= Exam.passingMarks
    - OR no exam defined for course

  ✓ Duration Requirement:
    - Current Date >= Batch.schedule.endDate

When all conditions met:
  1. Mark batch enrollment status: 'completed'
  2. Set User.lifecycleStatus: 'completed'
  3. Create Certificate record
  4. Generate PDF and QR code
  5. Return certificate details
